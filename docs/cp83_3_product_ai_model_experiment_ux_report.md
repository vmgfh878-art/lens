# CP83-3-P AI 모델 이전 실험 UX 강제 정리 보고서

## 1. 이전 실험 기본 숨김 방식

- `AI 모델` 화면의 기본 상태에서는 이전 실험 목록을 바로 펼치지 않는다.
- 기본 화면에는 아래만 보인다.
  - 제품 모델 현황 4슬롯
  - 선택한 제품 모델 상세
  - 접힌 실험 버튼 2개
    - `예측선 실험 보기`
    - `밴드 실험 보기`
- 브라우저 검증에서 초기 진입 시 열린 실험 목록은 0개였다.

## 2. 예측선 실험/밴드 실험 그룹 방식

- 이전 실험은 두 그룹만 사용한다.
  - 예측선 실험
  - 밴드 실험
- 기존처럼 `품질 기준 미통과`, `NaN 실패`, `Legacy`, `Composite`, `기타`를 별도 그룹으로 나누지 않는다.
- 기준 미달 여부는 실험 row의 작은 태그로만 표시한다.

## 3. 숨긴 항목

- NaN 실패
  - API 조회 대상에서 제외했다.
  - 기본 UI에 표시하지 않는다.
- composite
  - `isLegacyRun()` 기준으로 필터링해 이전 실험 목록에서도 제외했다.
  - 제품 UI 기본 노출 없음.
- 계산 불가
  - 실험 상세를 먼저 조회한 뒤 목표 대비 카드가 2개 이상 생성되는 실험만 목록에 표시했다.
  - 목표/실제/차이 카드가 부족한 실험은 목록에서 숨긴다.

## 4. 실험 이름 생성 기준

run_id는 기본 목록 이름으로 쓰지 않는다.

예측선 실험 이름은 아래 요소를 우선 사용한다.
- 모델 구조
- horizon
- patch_len
- stride / patch_stride
- feature_set

밴드 실험 이름은 아래 요소를 우선 사용한다.
- 모델 구조
- q_low / q_high
- feature_set
- horizon

현재 표시 예:
- `PatchTST h5 긴 패치 예측선`
- `PatchTST h5 dense overlap 예측선`
- `PatchTST h20 예측선 실험`
- `CNN-LSTM q15 AI 밴드`
- `CNN-LSTM 가격·변동성·거래량 밴드`

## 5. 이전 실험 클릭 시 상세 패널 변경 방식

- 이전 실험 row를 클릭하면 `selected` 상태가 `{ kind: "experiment", runId, category }`로 바뀐다.
- 같은 메인 상세 패널에서 `ExperimentDetail`을 렌더링한다.
- 제품 모델, 예측선 실험, 밴드 실험이 모두 같은 상세 패널 위치를 사용한다.
- 실험 상세에는 아래를 표시한다.
  - 실험 이름
  - 역할
  - 실험에서 바꾼 것
  - 판정
  - 왜 제품 모델이 아닌가
  - 목표 대비 평가 카드
  - 다음 개선 방향
  - 전문가용 상세 접기

## 6. 목표 대비 카드가 없는 실험 처리 방식

- `fetchAiRun()`으로 실험 상세를 먼저 조회한다.
- `buildExperimentCards()`로 카드 생성 가능 여부를 확인한다.
- 카드가 2개 미만이면 실험 목록에 표시하지 않는다.
- 화면에는 `계산 불가` 카드를 만들지 않는다.

## 7. 실패 이유 자연어 설명 방식

metric으로 확인 가능한 이유만 사용했다.

예측선 실험:
- 순위 상관이 0 이하이면 수익 방향 구분 약함으로 설명.
- 위험 오판율이 25%를 넘으면 위험 구간 오판이 많다고 설명.
- 큰 하락 포착률이 70% 미만이면 큰 하락 포착 부족으로 설명.

밴드 실험:
- 실제 포함률이 65% 미만이면 위험 범위를 충분히 덮지 못했다고 설명.
- 하단 이탈률이 15%를 넘으면 하방 위험을 충분히 덮지 못했다고 설명.
- 밴드 폭 반응도가 0 이하이면 변동성 반응 부족으로 설명.

확인 불가능한 원인은 만들지 않고 `실험 조건 상세는 전문가용 정보에서 확인`으로 남겼다.

## 8. 남은 아쉬운 점

- 실험 이름은 config 기반 휴리스틱이므로 모든 실험의 차이를 완벽히 설명하지는 않는다.
- 예측선 실험 중 오래된 metric 구조는 false_safe_tail_rate 같은 최신 지표가 없을 수 있어 카드 수가 줄어든다.
- 실패 이유 자연어 설명은 아직 완전한 리포트 엔진이 아니라 주요 metric 기준의 1차 설명이다.
