# CP122-UXR 백테스트 전략 신호 화면 UX 리뷰

## 1. 핵심 결론

현재 백테스트 화면의 전략 신호 구조는 방향이 좋다. 사용자가 티커를 직접 떠올리지 않아도 "오늘 볼 만한 종목"에서 시작해 단일 티커 백테스트로 내려갈 수 있게 만든 점은 제품 가치가 분명하다. 특히 상단 요약, 4개 신호 그룹, 카드 클릭으로 상세 백테스트를 바꾸는 흐름은 초안으로 충분히 설득력이 있다.

다만 지금 구조는 **백테스트 화면 안에 전략 스캐너가 강하게 들어온 상태**다. 현재 scanner가 12개 subset이라 데스크톱에서는 버틸 수 있지만, 구현상 그룹당 최대 10개 카드를 모두 노출하고(`frontend/src/components/BacktestView.tsx:1164-1168`), 모바일에서는 4개 그룹이 1열로 길게 쌓인다(`frontend/src/app/globals.css:3779-3781`). 향후 100개 또는 500개 universe로 확대되면 사용자는 상세 백테스트에 도달하기 전에 신호 카드 목록에서 지칠 가능성이 높다.

가장 중요한 UX 수정 방향은 **전략 신호를 "요약형 선택 레이어"로 줄이고, 선택 후 바로 백테스트 상세로 이어지는 감각을 강화하는 것**이다. 매수 후보와 위험 확대는 앞에 두되 과신을 막는 문구를 붙이고, 관망은 기본 접힘 또는 소형 리스트로 낮추는 편이 좋다.

정적 코드 기준으로 리뷰했다. 코드, 스타일, DB/API/schema, 모델 학습/inference는 실행하거나 수정하지 않았다.

## 3. 추천 레이아웃

추천 구조는 **요약 → 우선순위 그룹 → 선택 티커 백테스트 → 전략 설명** 순서다.

1. 상단 헤더는 현재처럼 유지하되 제목을 "전략 신호와 백테스트"로 바꾼다.
2. 요약 패널에는 12개 subset 한계, 기준일, 신호 확인 수, 선택 티커를 함께 보여준다.
3. 신호 그룹은 4열 카드 보드보다 "우선순위 섹션"으로 압축한다.
4. 기본 순서는 `진입 검토`, `위험 점검`, `전략 유지`, `신호 약함`이 좋다.
5. 각 그룹은 기본 노출 수만 보여주고, 초과분은 "더 보기"로 연다.
6. 관망은 기본 접힘으로 두고, "신호 약함 7개"처럼 count 중심으로 보여준다.
7. 카드 클릭 시 "AAPL 백테스트로 이동" 같은 버튼/CTA를 명시한다.
8. 클릭 후 아래 상세 백테스트 영역으로 자동 스크롤하거나, 상세 영역 상단에 "선택: AAPL, 진입 검토" sticky label을 둔다.
9. 직접 티커 입력은 상단 요약 내부의 compact input으로 일부 노출하고, 현재처럼 보조 패널도 유지한다.
10. 전략 설명과 검증 수치는 상세 아래에 두거나 접힘으로 둔다. 처음 진입 사용자에게는 신호 선택과 상세 백테스트가 먼저다.

모바일 레이아웃은 더 강하게 줄여야 한다.

1. 상단 요약 1개.
2. 그룹 탭 또는 아코디언.
3. 기본으로 `진입 검토`와 `위험 점검`만 펼침.
4. 각 그룹 카드 2~3개.
5. 직접 티커 입력은 항상 상단 근처에 유지.
6. 상세 백테스트는 선택 직후 바로 이어지게 배치.

## 4. 그룹별 노출 개수 제안

현재 12개 subset 기준:

| 그룹 | 기본 노출 | 더 보기 후 최대 | 기본 상태 | 이유 |
|---|---:|---:|---|---|
| 진입 검토 | 5개 | 10개 | 펼침 | 사용자의 "오늘 볼 만한 종목" 기대와 가장 직접적으로 맞다. |
| 위험 점검 | 5개 | 10개 | 펼침 | 리스크 관리 가치가 커서 매수 후보와 같은 우선순위가 적절하다. |
| 전략 유지 | 3개 | 8개 | 펼침 또는 축약 | 이미 보유 중인 전략 상태라 신규 탐색보다는 우선순위가 낮다. |
| 신호 약함 | 3개 | 10개 | 기본 접힘 | 정보 가치는 있지만 화면을 지배하면 제품 가치가 흐려진다. |

향후 100개 이상 확장 기준:

| 그룹 | 기본 노출 | 더 보기 단위 | 비고 |
|---|---:|---:|---|
| 진입 검토 | 5개 | 10개씩 | 정렬 기준을 예측선, 밴드 폭, 최근 추세 중 하나로 명시해야 한다. |
| 위험 점검 | 5개 | 10개씩 | 하방 위험 큰 순 또는 밴드 확장 큰 순 정렬이 필요하다. |
| 전략 유지 | 3개 | 10개씩 | 사용자가 보유 ticker를 따로 입력할 수 있게 직접 검색과 연결한다. |
| 신호 약함 | 0~3개 | 10개씩 | 기본 접힘이 맞다. count와 대표 3개만 보여준다. |

## 5. 문구/용어 제안

현재 용어는 직관적이지만, "매수 후보"가 투자 권유처럼 보일 수 있다. 제품 톤은 "추천"보다 "검토/점검/백테스트 확인" 쪽이 더 안전하다.

추천 용어:

| 현재 | 제안 | 이유 |
|---|---|---|
| 매수 후보 | 진입 검토 | 행동 지시 느낌을 줄이고 백테스트 확인 전 단계로 보인다. |
| 위험 확대 | 위험 점검 | 공포감을 낮추면서 리스크 관리 목적은 유지한다. |
| 보유 유지 | 전략 유지 | 실제 보유 종목이 아니라 전략상 position일 수 있어 오해를 줄인다. |
| 관망 | 신호 약함 | "관망하라"는 행동 조언보다 데이터 상태 설명에 가깝다. |
| 오늘의 신호 요약 | 오늘 볼 만한 전략 신호 | 제품 가치가 더 직접적이다. |
| 스캔 범위 12개 | 대표 12종목 데모 스캔 | 전체 universe가 아니라는 한계가 더 명확하다. |
| 실험 전략 v1 | 실험 전략, 투자 권유 아님 | 과신 방지 문구가 더 강해진다. |

상단 설명문 추천:

```text
대표 12종목만 대상으로 Lens Balance v1의 최신 전략 상태를 요약합니다.
전체 500티커 스캐너가 아니며, 각 카드는 단일 티커 백테스트를 빠르게 열기 위한 탐색 신호입니다.
```

카드 CTA 추천:

```text
백테스트 보기
```

선택 후 상세 영역 제목 추천:

```text
선택한 신호: AAPL · 진입 검토
아래 결과는 해당 티커의 과거 1D 백테스트입니다.
```

## 6. 다음 CP에서 수정할 액션 리스트

P1 액션:

1. 그룹별 기본 노출 개수를 줄이고 더 보기/접기 UI를 추가한다.
2. 관망 그룹은 기본 접힘 또는 count 요약으로 바꾼다.
3. 카드에 "백테스트 보기" CTA를 넣고, 클릭 후 상세 영역으로 이어지는 시각적 연결을 만든다.
4. 상단에 "대표 12종목 데모 스캔, 전체 universe 아님" 고지를 명확히 추가한다.
5. 모바일에서 신호 그룹을 아코디언 또는 탭 구조로 줄인다.

P2 액션:

1. 용어를 `진입 검토 / 위험 점검 / 전략 유지 / 신호 약함`으로 낮춰 투자 권유 인상을 줄인다.
2. 화면 제목을 "전략 신호와 백테스트"처럼 백테스트 화면 정체성과 맞춘다.
3. 직접 티커 입력을 상단 요약 근처에도 compact하게 배치한다.
4. 그룹 정렬 기준을 문구로 표시한다.
5. empty state는 반복 카드 대신 그룹 축약 메시지로 처리한다.

P3 액션:

1. 카드의 긴 reason은 2줄 유지하되, 한 줄 핵심 근거를 별도 표시한다.
2. 선택된 카드와 상세 제목의 ticker/group을 같은 색 또는 badge로 연결한다.
3. 전략 설명과 검증 수치는 기본 펼침보다 하단 접힘으로 낮추는 것을 검토한다.

확인: 이번 리뷰에서는 `frontend/src/components/BacktestView.tsx`, `frontend/src/app/globals.css`, `frontend/src/components/AppShell.tsx`를 읽기 전용으로 확인했고, 작성한 파일은 이 보고서뿐이다.
