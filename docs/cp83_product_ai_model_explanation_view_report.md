# CP83-P AI 모델 설명 화면 재구성 보고서

## 1. 화면 목적 변경 내용

- 사이드바 사용자 노출 메뉴를 `모델 학습`에서 `AI 모델`로 변경했다.
- 화면 제목도 `AI 모델`로 변경했다.
- 기존 run 콘솔/후보 선택 중심 흐름을 `현재 제품에 쓰이는 AI 모델을 설명하는 화면`으로 바꿨다.
- 기본 설명 문구:
  - `Lens는 예측선 모델과 AI 밴드 모델을 분리해서 평가하고, 검증된 모델만 주식 보기 화면에 사용합니다.`

## 2. 4개 제품 슬롯 구조

상단 `제품 모델 현황` 섹션에 4개 고정 슬롯을 배치했다.

| 슬롯 | 상태 | 모델 | 버전 | run_id 표시 정책 |
| --- | --- | --- | --- | --- |
| 1D 보수적 예측선 | 사용 중 | PatchTST | v1 | 기본 카드에는 숨김, 전문가용 상세에만 표시 |
| 1D AI 밴드 | 사용 중 | CNN-LSTM | v1 | 기본 카드에는 숨김, 전문가용 상세에만 표시 |
| 1W 보수적 예측선 | 준비 중 | 제품 후보 없음 | 없음 | 노출 없음 |
| 1W AI 밴드 | 준비 중 | 제품 후보 없음 | 없음 | 노출 없음 |

## 3. 1D 보수적 예측선 v1 설명

- 대상 run: `patchtst-1D-efad3c29d803`
- 기본 화면 이름: `1D 보수적 예측선 v1`
- 한 줄 판정:
  - `수익 방향과 종목 순위 판단에는 사용할 수 있지만, 위험 회피 품질은 개선 중입니다.`
- 모델 설명:
  - PatchTST 기반 모델
  - 최근 252거래일 데이터를 보고 앞으로 5거래일 수익 방향을 예측
  - 하방 위험을 더 조심스럽게 보도록 학습한 보수적 예측선
- 사용 데이터:
  - 가격
  - 거래량
  - 기술적 지표
  - 재무·거시·시장 폭 관련 피처
- 목표 대비 카드:
  - 방향/순위 예측
  - 상위/하위 구분
  - 위험 오판
  - 큰 하락 포착
  - 수수료 반영 안정성

## 4. 1D AI 밴드 v1 설명

- 대상 run: `cnn_lstm-1D-d0c780dee5e8`
- 기본 화면 이름: `1D AI 밴드 v1`
- 한 줄 판정:
  - `1D 위험 범위 보조지표로 사용할 수 있는 제품 후보입니다.`
- 모델 설명:
  - CNN-LSTM 기반 모델
  - 최근 60거래일 가격·변동성·거래량 정보를 사용
  - 앞으로 5거래일의 예상 변동 범위를 계산
- 사용 데이터:
  - 가격
  - 변동성
  - 거래량
- 목표 대비 카드:
  - 포함률
  - 상단/하단 균형
  - 밴드 폭
  - 변동성 반응
  - 하방 위험 반응

## 5. 1W 준비 중 처리

- 1W 보수적 예측선은 `준비 중`으로 표시한다.
- 1W AI 밴드도 `준비 중`으로 표시한다.
- 오류처럼 보이지 않게 설명을 별도 준비 상태로 정리했다.
- 1W 슬롯에는 run_id나 내부 모델 후보 ID를 노출하지 않는다.

## 6. 이전 실험/실패 실험 표시 정책

- 제품 슬롯 아래에 `이전 실험` 섹션을 분리했다.
- 제품 후보 run과 이전 실험 run이 같은 목록에서 섞이지 않도록 했다.
- composite run은 `Composite 결합 실험`으로 표시하며 현재 제품 기본에서 제외됨을 설명한다.
- `failed_quality_gate`는 `품질 기준 미통과 실험`으로 분류한다.
- `failed_nan`은 `시스템 오류성 실험 보기` 접기 영역에만 넣어 기본 화면에서 과하게 노출되지 않게 했다.

## 7. NaN 실패 숨김 정책

- NaN 실패 실험은 삭제하지 않았다.
- 기본 제품 모델 현황과 이전 실험 기본 목록에는 직접 노출하지 않는다.
- 사용자가 원하면 접기 영역에서 확인할 수 있다.

## 8. 평가 테이블을 목표 대비 카드로 바꾼 방식

- 기존 평가 테이블/metric dump 중심 UI를 기본 화면에서 제거했다.
- 비전문가가 읽기 쉬운 목표 대비 카드로 바꿨다.
- raw metric key는 기본 화면에 두지 않고 전문가용 상세에만 남겼다.

## 9. 전문가용 상세 접기 영역

전문가용 상세에는 아래 정보를 유지했다.

- 실행 ID
- 모델 이름
- 역할
- 사용 데이터
- 모델 선택 기준
- 예측 기간
- 입력 길이
- PatchTST 설정
- CNN-LSTM 밴드 설정
- 실험 추적 상태
- 체크포인트 경로
- 주요 val/test metric

## 10. 남은 아쉬운 점

- 제품 슬롯 값 일부는 명시된 제품 후보 run_id를 기준으로 보여주며, 모든 자연어 판정이 DB에서 자동 계산되는 구조는 아니다.
- 이전 실험의 실패 이유는 아직 완전한 원인 분석 자동화가 아니라 상태와 모델 종류 기반의 안전한 설명이다.
- raw metric 전체 탐색/검색 기능은 없다.
