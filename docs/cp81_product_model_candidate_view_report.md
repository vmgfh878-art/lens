# CP81-P 모델 후보 화면 재구성 및 차트 레이어 토글 정리

CP81-P는 모델 학습 화면을 학습 로그 콘솔이 아니라 제품 후보 관리 화면으로 바꾸고, 주식 보기 차트의 가격 오버레이·거래량·하단 지표 토글을 분리한 작업이다.

## 1. 차트 레이어/거래량/하단 지표 토글 구조

- 주식 보기 우측 패널의 레이어 제어를 세 그룹으로 정리했다.
- `가격 오버레이`: 예측선, AI 밴드, 보수적 기준선
- `거래량`: 거래량 bar
- `하단 지표`: RSI, MACD, ATR, AI 밴드 폭, 거래량 변화 후보
- 1D/1W/1M 전환 시 기존 토글 상태를 유지하도록 했다.
- 1M은 기존 계약대로 가격 전용이며, AI overlay는 표시하지 않는다.

## 2. 거래량 bar ON/OFF 구현 내용

- `frontend/src/components/StockView.tsx`
  - `layers.volumeBar` 상태를 추가했다.
  - 기본값은 `true`로 두었다.
  - 거래량 데이터가 없으면 토글을 비활성화한다.
- `frontend/src/components/Chart.tsx`
  - 거래량 histogram series 생성 조건을 `layers.volumeBar`와 실제 거래량 데이터 존재 여부로 제한했다.
  - 거래량 legend도 거래량 bar가 켜져 있고 데이터가 있을 때만 표시한다.

## 3. 기본 하단 지표 변경 내용

- 기본 하단 지표는 `RSI`, `MACD`, `AI 밴드 폭`으로 유지했다.
- `거래량 변화`는 거래량 bar가 생겼으므로 기본 표시에서는 제외했다.
- 다만 완전히 삭제하지 않고 하단 지표 선택 후보에는 남겨 두었다.
- ATR은 기존 정책대로 실제 값이 있을 때만 선택 가능하다.

## 4. 모델 후보 화면 구조 변경 내용

- `frontend/src/components/TrainingView.tsx`
  - 상단 제목을 `모델 후보`로 정리했다.
  - 설명 문구를 추가했다.
    - `Lens는 예측선 모델과 AI 밴드 모델을 따로 평가하고, 제품 후보만 기본 화면에 사용합니다.`
  - 후보 목록을 역할별로 분리했다.
    - 예측선 후보
    - AI 밴드 후보
    - 이전 실험 / 기타
    - 실패한 실험
  - 제품 후보 카드 영역을 추가해 예측선 후보와 AI 밴드 후보를 나란히 확인할 수 있게 했다.
  - 선택한 run의 실행 정보, 파라미터 요약, 품질 지표, 평가 테이블은 아래 상세 영역에서 확인하도록 유지했다.

## 5. 실패 run 기본 숨김/접기 정책

- 기본 화면에는 completed 제품 후보를 우선 노출한다.
- `failed_quality_gate`와 `failed_nan`은 상단 탭에서 제거했다.
- 실패 run은 `실패한 실험 보기` 접기 영역에서만 확인할 수 있다.
- composite와 legacy run은 `이전 실험 보기` 접기 영역으로 내렸다.
- 실패와 legacy 산출물은 삭제하지 않았고, 제품 기본 후보처럼 보이지 않게만 정리했다.

## 6. 예측선 후보 상세 표시 내용

예측선 후보 카드에는 아래 내용을 표시한다.

- 모델 버전명: 예측선 후보
- 모델 구조: PatchTST
- 예측 기간: 5거래일
- 사용 데이터: 전체 피처
- 패치 길이
- 패치 간격
- 입력 길이
- 체크포인트 선택
- 상태: 예측선 후보
- 실험 추적
- 순위 상관(IC)
- 상위-하위 수익 차
- 위험 오판율
- 큰 하락 포착률
- 수수료 반영 샤프

설명 문구:

`향후 수익 방향과 순위 판단을 돕는 모델입니다. 현재 위험 오판율은 개선 중이므로 단독 매매 신호가 아니라 참고선으로 사용합니다.`

## 7. AI 밴드 후보 상세 표시 내용

AI 밴드 후보 카드에는 아래 내용을 표시한다.

- 모델 버전명: AI 밴드 후보
- 모델 구조: CNN-LSTM
- 예측 기간: 5거래일
- 사용 데이터: 가격·변동성·거래량
- 입력 길이
- 밴드 분위수
- 밴드 손실 가중치
- 밴드 방식
- 체크포인트 선택
- 상태: AI 밴드 후보
- 목표 포함률
- 실제 포함률
- 포함률 오차
- 하단 이탈률
- 상단 이탈률
- 평균 밴드 폭
- 비대칭 interval score
- 변동성 반응도

설명 문구:

`예상 변동 범위를 보여주는 모델입니다. 밴드가 넓어지는 구간은 모델이 더 큰 변동 가능성을 보는 구간입니다.`

## 8. 상세 정보 접기 영역에 남긴 내부 용어

- 기본 후보 카드에서는 내부 용어를 최소화했다.
- 선택 run 상세 영역에는 검토용으로 아래 내부 정보를 남겼다.
  - 실행 ID
  - role을 한글화한 역할
  - feature_set을 한글화한 사용 데이터
  - checkpoint_selection
  - wandb_status를 한글화한 실험 추적
  - checkpoint path 존재 여부
- raw JSON 형태의 값은 기본 노출하지 않았다.
- 로컬 로그 파일은 프론트에서 직접 읽지 않고, `읽기 전용 로그 API 필요` 상태로 표시했다.

## 9. 레이아웃 overflow 수정 내용

- 긴 run_id는 `run-row__id`에서 ellipsis 처리되도록 했다.
- run row grid 폭을 조정해 카드 안 텍스트가 튀어나오는 현상을 줄였다.
- metric card 값, 설정 요약 값, 실행 정보 값에 줄바꿈 안전 처리를 추가했다.
- 후보 카드 설명 영역의 줄 높이와 최대 폭을 조정했다.
- 실패/이전 실험 접기 영역에 테두리와 여백을 부여해 기본 후보와 구분했다.

## 11. 남은 아쉬운 점

- 모델 후보 상세 패널은 아직 제품 후보 카드와 선택 run 상세 영역이 함께 있어 정보량이 많다.
- 추후에는 선택 후보 1개 중심의 상세 패널로 더 정돈할 수 있다.
- 거래량 bar OFF 상태는 차트 DOM보다는 시각 검증 중심이라 자동화 검증 정밀도는 더 보강할 수 있다.
- 로컬 학습 로그를 화면에 직접 연결하려면 별도 read-only 로그 API가 필요하다.
