# CP81-2-P 예측선 표시 통합 및 모델 후보 화면 단순화

CP81-2-P는 주식 보기 화면의 line layer를 하나의 `보수적 예측선`으로 통합하고, 모델 후보 화면을 상단 후보 선택과 하단 상세 설명 구조로 단순화한 작업이다.

## 1. 예측선/보수적 기준선 통합 방식

- 주식 보기 기본 화면에서 `예측선`과 `보수적 기준선`을 별도 레이어로 보여주지 않도록 정리했다.
- line layer는 화면에서 항상 `보수적 예측선` 하나로 표시한다.
- 차트 legend도 `보수적 예측선` 하나만 표시한다.
- 우측 가격 오버레이 토글도 `보수적 예측선`, `AI 밴드` 두 개만 남겼다.
- 기존 내부 필드명과 API/DB 값은 변경하지 않았다.

## 2. conservative_series와 line_series 선택 우선순위

- 표시 직전의 우선순위를 아래처럼 고정했다.
  1. `conservative_series`가 있으면 `conservative_series` 사용
  2. 없으면 `line_series` 사용
- rolling history도 같은 기준으로 첫 forecast 값을 선택한다.
- 값 자체를 수정하거나 보정하지 않았다.
- forecast date와 series 길이가 맞지 않거나 값이 유효하지 않으면 기존처럼 조용한 경고만 표시한다.

## 3. 차트 legend/toggle 변경 내용

- 제거:
  - `예측선` 별도 토글
  - `보수적 기준선` 별도 토글
  - `예측선` legend
  - `보수적 기준선` legend
- 유지/변경:
  - `보수적 예측선` 토글
  - `AI 밴드` 토글
  - `거래량 bar` 토글
  - `AI 위험 범위` legend
  - `과거 예측` legend
  - `오늘 이후 예측` legend
- 설명 문구는 `하방 위험을 더 조심스럽게 반영한 AI 예측선입니다.`로 정리했다.

## 4. 모델 후보 화면 구조 변경

- 기존 좌우 분리형 `역할별 후보 목록 + 제품 후보 카드 + 상세 패널` 구조를 줄였다.
- 새 구조:
  - 상단 `후보 선택`
  - 하단 `선택 후보 상세`
- 상단 후보 선택에는 아래 그룹을 둔다.
  - 예측선 후보
  - AI 밴드 후보
  - 이전 실험 보기
  - 실패 실험 보기
- 제품 후보 카드 영역은 제거했다.
- 선택한 후보 하나의 설명, 요약, 지표, 판단 문구, 상세 정보를 하단에서 넓게 보여준다.

## 5. 후보 카테고리별 설명 문구

- 보수적 예측선 후보:
  - `향후 수익 방향을 참고하되, 하방 위험을 더 조심스럽게 보도록 학습한 예측선 후보입니다. 현재 수익/순위 예측력은 살아 있지만, 위험 오판율은 계속 개선 중입니다.`
- AI 밴드 후보:
  - `예상 변동 범위를 보여주는 AI 위험 범위 후보입니다. 밴드가 넓어지는 구간은 모델이 더 큰 변동 가능성을 보는 구간입니다.`
- 이전 실험:
  - `제품 후보로 쓰지는 않지만, 모델 구조와 실험 방향을 비교하기 위해 남겨둔 실행입니다. composite 실행은 제품 기본 후보가 아닙니다.`
- 실패 실험:
  - `품질 기준을 통과하지 못했거나 학습 중 오류가 발생한 실험입니다. 제품 후보에는 사용하지 않습니다.`

## 6. 제품 후보/이전 실험/실패 실험 구분 방식

- 제품 후보:
  - line model, band model completed 후보를 우선 노출한다.
  - `patchtst-1D-efad3c29d803`은 `보수적 예측선 모델 v1`로 표시한다.
  - `cnn_lstm-1D-d0c780dee5e8`은 `AI 밴드 모델 v1`로 표시한다.
- 이전 실험:
  - composite/legacy/기타 completed 후보는 `이전 실험 보기` 접기 영역으로 내렸다.
  - composite는 제품 후보가 아니라고 상세 설명과 판단 문구에서 명확히 표시한다.
- 실패 실험:
  - `failed_quality_gate`, `failed_nan`은 `실패 실험 보기` 접기 영역에서만 선택할 수 있다.
  - 제품 후보에는 사용하지 않는다고 명시했다.

## 7. overflow 수정 내용

- 후보 선택을 chip 형태로 바꾸고 flex-wrap을 적용했다.
- 긴 실행 ID는 기본 chip에서 숨기고 상세 정보 접기 영역에만 남겼다.
- chip 내부 텍스트는 ellipsis 처리했다.
- 상세 설명은 별도 문단으로 두고, metric grid와 분리해 줄바꿈이 자연스럽게 되도록 했다.
- 좁은 화면에서는 후보 chip과 품질 지표 grid가 1열로 내려가도록 했다.

## 8. 남은 아쉬운 점

- 후보 선택 영역의 세부 인터랙션은 아직 탭보다는 chip/접기 조합에 가깝다.
- 상세 정보 접기 안의 내부 용어는 검토 편의상 남아 있다.
- 실패 이유를 더 예쁘게 요약하려면 run meta나 평가 로그를 읽는 별도 read-only API가 필요하다.
- 모델 후보 화면의 모바일 실제 스크린샷 검증은 이번 CP에서 별도 캡처까지 진행하지 않았다.
