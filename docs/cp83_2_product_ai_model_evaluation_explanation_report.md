# CP83-2-P AI 모델 화면 단순화 및 목표 대비 평가 카드 보강 보고서

## 1. 제품 모델 현황 4슬롯 단순화 내용

- 상단 `제품 모델 현황`은 4개 제품 슬롯만 보이도록 유지했다.
- 슬롯 표시 정보는 이름, 상태, 버전, 간단한 역할 중심으로 정리했다.
- 기본 노출 슬롯:
  - `1D 보수적 예측선 / v1 / 사용 중`
  - `1D AI 밴드 / v1 / 사용 중`
  - `1W 보수적 예측선 / 준비 중`
  - `1W AI 밴드 / 준비 중`
- run_id, feature_set, 내부 상태값은 기본 슬롯에서 노출하지 않는다.

## 2. 제거한 문구

기본 제품 UI에서 아래 성격의 문구를 제거하거나 축소했다.

- `제품 후보` 과다 표현
- `제품 후보와 이전 실험은 분리해서 봅니다`
- `시스템 오류성 실험 보기`
- composite 관련 기본 노출 문구
- 품질 미달/실패를 별도 큰 섹션으로 나누는 문구

## 3. 이전 실험 섹션 통합 방식

- 기존 `이전 실험`, `품질 기준 미통과 실험`, `NaN 실패` 분리 구조를 하나의 `이전 실험` 섹션으로 통합했다.
- 각 실험은 작은 태그로만 구분한다.
  - `제품 미사용`
  - `기준 미달`
  - `보류`
  - `개선 전 버전`
- 제품 모델 현황보다 시각적으로 작게 보이도록 섹션을 낮은 위계로 유지했다.

## 4. NaN 실패 숨김 정책

- NaN 실패 run은 제품 UI 기본 화면에서 조회/표시하지 않는다.
- NaN 실패는 제품 모델 설명 화면의 사용자 맥락에서는 시스템 오류성 실험이므로 기본 노출에서 제외했다.
- 필요 시 리포트나 별도 운영 점검에서만 다루는 것이 맞다.

## 5. composite UI 제거 방식

- composite/legacy run은 이전 실험 목록에서도 기본 표시하지 않도록 필터링했다.
- 기존 composite run은 삭제하지 않았다.
- readiness에서는 여전히 `LEGACY_OK`로 보존 여부를 확인한다.
- 제품 UI에서는 line/band 제품 모델과 섞이지 않도록 노출하지 않는다.

## 6. 목표 대비 평가 카드 구조

평가 카드는 아래 구조로 통일했다.

- 지표 이름
- 목표치
- 실제값
- 차이
- 판정
- 한 줄 해석

기본 화면에서는 raw metric key를 먼저 보여주지 않는다.
전문가용 상세 접기 영역에서만 raw key/value를 확인할 수 있다.

## 8. 이전 실험 표시 방식

- 이전 실험은 run_id 대신 읽기 쉬운 이름을 사용한다.
  - `PatchTST h20 예측선 실험`
  - `PatchTST 예측선 실험`
  - `CNN-LSTM 밴드 초기 실험`
- 실험 상세에서도 목표 대비 평가 카드를 사용한다.
- 실험 조건을 정확히 추측하지 않고, 모르는 항목은 전문가용 상세에서 확인하도록 안내한다.

## 9. 전문가용 상세 접기 유지 내용

전문가용 상세에는 아래 정보를 유지했다.

- 실행 ID
- 모델 이름
- 역할
- 사용 데이터
- 모델 선택 기준
- 예측 기간
- 입력 길이
- patch_len / stride
- q_low / q_high
- lambda_band
- band_mode
- 실험 추적 상태
- 체크포인트 경로
- raw metric key/value

## 10. 남은 아쉬운 점

- 이전 실험 이름은 아직 run config를 완전히 해석해서 자동 생성하지는 않는다.
- 이전 실험의 실패 이유도 모든 metric을 종합한 자연어 자동 진단까지는 아니다.
- 제품 모델별 목표치는 현재 제품 판단 기준을 화면에 명시한 것이며, 별도 설정 API로 관리되지는 않는다.
