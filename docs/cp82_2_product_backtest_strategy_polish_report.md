# CP82-2-P 백테스트 데모 전략 교체 및 소형 UX 수정

CP82-2-P는 단일 티커 백테스트 화면의 기본 데모 전략을 `Risk Guard v1`에서 `Trend Guard v1`로 바꾸고, 거래비용 박스와 티커 검색 후보 UI를 작게 다듬은 작업이다.

## 1. Risk Guard v1이 데모 전략으로 약했던 이유

- `Risk Guard v1`은 보수적 예측선이 양수이고 AI 밴드 하단도 -3%보다 깊지 않을 때만 보유하는 강한 방어형 규칙이다.
- AAPL/MSFT처럼 장기 상승 흐름이 있는 종목에서는 현금 대기가 과도하게 늘어날 수 있다.
- Phase 1 데모에서는 “위험 구간을 줄이는 방식”을 보여주되, 지나치게 시장 참여가 줄어드는 전략은 제품 신뢰도를 해칠 수 있다고 판단했다.

## 2. Trend Guard v1 규칙

목적:

- 상승 추세는 따라가되, AI 밴드와 RSI로 과한 위험 구간만 줄이는 단일 티커 추세 방어 전략

사용 지표:

- 가격
- 60일 이동평균 또는 `ma_60_ratio`
- 보수적 예측선
- AI 밴드 하단
- RSI

진입/보유 조건:

- 가격이 60일 이동평균 위 또는 `ma_60_ratio >= 0`
- 보수적 예측선 >= -1%
- AI 밴드 하단 > -5%
- RSI < 82

청산/회피 조건:

- 가격이 60일 이동평균 아래 또는 `ma_60_ratio < 0`
- 또는 AI 밴드 하단 <= -7%
- 또는 RSI >= 88

포지션:

- 단일 티커 100%
- 현금 100%
- 부분 비중 없음
- 포트폴리오 없음

## 3. 기존 Risk Guard v1 처리 방식

- `Risk Guard v1`은 삭제하지 않고 전략 선택 후보에 남겼다.
- 기본 선택은 `Trend Guard v1`로 변경했다.
- Risk Guard 설명에는 `강한 방어형 전략` 성격을 명시했다.

## 4. 신호 생성 방식

- 신호 날짜는 prediction `asof_date` 기준이다.
- 보수적 예측선은 `conservative_series` 우선, 없으면 `line_series`를 사용한다.
- AI 밴드 하단은 horizon 내 `lower_band_series` 최저값을 사용한다.
- 보수적 예측선 수익률과 AI 밴드 하단 수익률은 해당 날짜 종가 대비 비율로 계산한다.
- RSI는 indicator row를 사용하며 0~1 값이면 0~100으로 변환한다.
- `ma_60_ratio`가 있으면 우선 사용하고, 없으면 과거 가격 60개로 60일 이동평균 괴리를 프론트에서 계산한다.
- prediction이 없는 날짜에는 새 신호를 만들지 않고 직전 신호를 유지한다.

## 5. lookahead leakage 방지 유지 여부

- 신호 생성에는 `asof_date`에 존재하는 가격, RSI, prediction row만 사용한다.
- 미래 실현 수익률은 신호 생성에 사용하지 않는다.
- 60일 이동평균 fallback도 해당 날짜까지의 과거 가격만 사용한다.
- 신호는 다음 가격 구간의 포지션에 반영한다.

## 6. 거래비용 박스 축소 내용

- 기존 metric card 크기의 거래비용 박스를 작은 `fee-pill`로 바꿨다.
- `거래 비용 / 10bp / 수수료 반영`을 보조 정보처럼 보이게 했다.
- 핵심 성과 카드와 같은 위계로 보이지 않도록 배경과 패딩을 줄였다.

## 7. 티커 검색 드롭다운 수정 내용

- 검색 후보 영역에 `margin-top`, `padding-top`, `z-index`를 추가해 입력창과 겹치지 않게 했다.
- 후보 버튼에 `max-width`, `ellipsis`를 적용해 긴 ticker/sector 텍스트가 넘치지 않게 했다.
- AAPL/MSFT 검색 후보가 입력창 아래에 표시되는지 확인했다.

## 8. AAPL/MSFT 확인 결과

- AAPL:
  - 기본 전략이 `Trend Guard v1`로 표시됨
  - 전략 설명 패널 표시됨
  - 전략 수익률, 단순 보유 수익률, 초과 수익률, 최대 낙폭, 현금 대기 비율, 거래 횟수 표시됨
  - 가격 차트와 전략 vs 단순 보유 차트 표시됨
- MSFT:
  - 직접 입력 조회 동작
  - `Trend Guard v1` 결과 표시
  - 가격 차트와 핵심 지표 표시
- AMP:
  - 가격/RSI는 표시
  - prediction row 없음 상태 표시
  - fake prediction 생성 없음

## 9. 남은 백테스트 개선점

- 전략 계산은 아직 프론트 계산이다. 추후 read-only 서버 계산 API와 수치 일치 테스트가 필요하다.
- 차트는 단순 SVG 기반이라 전문 백테스트 차트 수준은 아니다.
- 전략 파라미터를 사용자가 직접 조정하는 UI는 아직 없다.
- 1W/1M 전략 계산은 아직 제외했다.
- 손실 회피율 등 일부 지표는 더 명확한 정의 문구를 추가할 여지가 있다.
