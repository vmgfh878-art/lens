# CP122-2-P 백테스트 전략 신호 UX 압축 수정 보고서

CP122-2-P는 CP122에서 만든 전략 신호 화면을 더 압축하고, 종목 카드에서 단일 티커 상세 백테스트로 이어지는 연결감을 보강한 작업이다.

## 1. 전략 신호 기본 노출 축소

그룹별 기본 노출 개수를 줄였다.

| 그룹 | 기본 노출 |
| --- | --- |
| 매수 후보 | 3개 |
| 보유 유지 | 3개 |
| 위험 확대 | 3개 |
| 관망 | 기본 접힘 |

펼쳐도 각 그룹은 최대 10개까지만 표시한다.

## 2. 더 보기 / 접기

- 각 그룹에 더 보기/접기 버튼을 추가했다.
- 보유 유지처럼 3개보다 많은 그룹은 `더 보기`로 확장된다.
- 확장 뒤에는 `접기`로 다시 기본 노출 상태로 돌아간다.
- 관망은 기본 접힘이고 `관망 보기` 버튼으로만 펼친다.

## 3. 관망 그룹 위계 낮추기

- 관망 그룹은 기본 카드 노출을 하지 않는다.
- 패널은 유지하되 dashed border와 약한 배경으로 다른 그룹보다 낮은 강조 스타일을 적용했다.
- 화면 상단을 과도하게 차지하지 않도록 접힌 안내만 표시한다.

## 4. 카드 클릭 후 상세 연결 보강

- 선택된 카드에 active 스타일을 추가했다.
- 선택 카드에는 왼쪽 accent bar와 배경 강조가 들어간다.
- 카드 클릭 또는 직접 입력 조회 후 상세 백테스트 영역으로 부드럽게 이동한다.
- 상세 영역 상단에 선택 ticker와 신호를 표시한다.
  - 예: `MSFT 단일 티커 상세`
  - 예: `전략상 보유 유지 · 기준일 2026-05-04`

## 5. subset 한계 고지

전략 신호 요약 아래에 아래 문구를 작은 보조 텍스트로 추가했다.

`현재는 12개 주요 종목 기준 전략 신호입니다. 전체 scanner API가 붙기 전까지는 subset 신호로만 봅니다.`

경고 배너로 만들지 않고 보조 문구로 처리했다.

## 6. 직접 티커 확인 영역 정리

- 직접 입력은 전략 신호 아래 보조 영역으로 유지했다.
- 문구는 `신호 목록에 없는 종목도 직접 확인할 수 있습니다.`로 줄였다.
- 입력창과 추천 목록 구조는 기존 겹침 방지 스타일을 유지했다.

## 8. 검증 결과

### 명령 검증

- `npm run build`: 통과
- `scripts/check_demo_readiness.ps1`: 통과
  - backend health OK
  - frontend OK
  - frontend-static OK
  - AAPL 가격 OK
  - product LM/BM run/history OK

### 브라우저 검증

- 백테스트 화면 진입 확인
- 기본 전략 신호 그룹 표시 확인
- subset 한계 문구 확인
- 기본 카드 개수 축소 확인
  - 보유 유지 기본 3개
  - 위험 확대 기본 3개
  - 관망 기본 접힘
- 보유 유지 `더 보기` 클릭 시 8개 표시 확인
- `접기` 클릭 시 3개로 복귀 확인
- MSFT 카드 클릭 시 상세 영역 전환 확인
- 선택 카드 active style 확인
- 상세 제목에 `MSFT 단일 티커 상세`과 신호 표시 확인
- 직접 티커 입력으로 `T` 조회 확인
- `T`는 prediction history가 없어 저장 결과 없음 상태로 표시 확인
- console error/warn 0건 확인

### 모바일/좁은 화면

- 실제 브라우저는 현재 데스크톱 폭에서 확인했다.
- CSS에서는 좁은 화면에서 signal board, summary, 상세 heading이 1열로 접히도록 responsive rule을 추가했다.
- 다음 실제 모바일 QA에서는 카드 높이와 더 보기 위치를 추가 확인하는 것이 좋다.

## 9. 남은 한계

- 아직 전체 500티커 scanner API가 없으므로 12개 subset 기반이다.
- 카드 클릭 후 scroll은 브라우저 기본 smooth scroll에 의존한다.
- 관망 그룹에 종목이 많아지는 경우에는 별도 필터나 정렬 기준이 필요할 수 있다.
